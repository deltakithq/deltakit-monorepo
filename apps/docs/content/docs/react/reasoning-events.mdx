This recipe shows how to stream and display reasoning/thinking content from AI models that expose their chain-of-thought process.

## Overview

Modern reasoning models (OpenAI's o1/o3, Moonshot's Kimi, etc.) generate internal "thinking" before producing final responses. Streaming this reasoning to users provides transparency and improves trust.

**Key Challenge**: Reasoning arrives as small text chunks (like `text_delta`), but should be displayed as a single accumulating block—not multiple separate boxes.

## The Pattern

```
Backend (SSE)          Frontend (useStreamChat)           UI
     │                         │                            │
     ├─ reasoning chunk ───────┤                            │
     ├─ reasoning chunk ───────┼─ accumulate ───────────────┤
     ├─ reasoning chunk ───────┤     (single part)          │
     │                         │                     ┌──────▼──────┐
     ├─ text_delta ────────────┼─ appendText()       │  Thinking   │
     │                         │                     │  •••        │
     └─ text_delta ────────────┤                     │  reasoning  │
                               │                     │  content    │
                               │                     └─────────────┘
```

## Backend Implementation

### Event Format

Send reasoning as SSE events with a `type` field:

```
data: {"type": "reasoning", "text": "Let me analyze..."}

data: {"type": "reasoning", "text": " the user's question."}

data: {"type": "text_delta", "delta": "Here's my answer..."}
```

### Deduplication Strategy

Reasoning models often emit overlapping chunks. Track what's been sent to avoid duplicates:

**Python/FastAPI Example:**

```python
@app.post("/chat")
async def chat(request: Request):
    runner = Runner.run_streamed(agent, input=request.message)
    
    async def event_generator():
        reasoning_buffer = ""
        
        async for event in runner.stream_events():
            # Handle reasoning deltas
            if event.type == "raw_response_event":
                data = event.data
                
                if "reasoning" in str(getattr(data, 'type', '')):
                    text = getattr(data, 'text', '') or getattr(data, 'delta', '')
                    
                    if text:
                        # Check for overlapping content
                        if text.startswith(reasoning_buffer) and len(text) > len(reasoning_buffer):
                            # Only emit the new portion
                            new_text = text[len(reasoning_buffer):]
                            yield f'data: {{"type": "reasoning", "text": {json.dumps(new_text)}}}\n\n'
                            reasoning_buffer = text
                        elif text not in reasoning_buffer:
                            # New content
                            yield f'data: {{"type": "reasoning", "text": {json.dumps(text)}}}\n\n'
                            reasoning_buffer = text
                
                elif hasattr(data, 'delta'):
                    yield f'data: {{"type": "text_delta", "delta": {json.dumps(data.delta)}}}\n\n'
    
    return StreamingResponse(event_generator(), media_type="text/event-stream")
```

**Node.js/Express Example:**

```typescript
app.post('/chat', async (req, res) => {
  res.setHeader('Content-Type', 'text/event-stream');
  res.setHeader('Cache-Control', 'no-cache');
  
  let reasoningBuffer = '';
  
  for await (const event of streamEvents()) {
    if (event.type === 'reasoning') {
      const text = event.text || event.delta || '';
      
      // Deduplicate overlapping chunks
      if (text.startsWith(reasoningBuffer) && text.length > reasoningBuffer.length) {
        const newText = text.slice(reasoningBuffer.length);
        res.write(`data: ${JSON.stringify({ type: 'reasoning', text: newText })}\n\n`);
        reasoningBuffer = text;
      } else if (!reasoningBuffer.includes(text)) {
        res.write(`data: ${JSON.stringify({ type: 'reasoning', text })}\n\n`);
        reasoningBuffer = text;
      }
    } else if (event.type === 'text_delta') {
      res.write(`data: ${JSON.stringify({ type: 'text_delta', delta: event.delta })}\n\n`);
    }
  }
  
  res.write('data: [DONE]\n\n');
  res.end();
});
```

## Frontend Implementation

### 1. Define Custom Event Types

```tsx
import { useStreamChat, type ContentPart } from "@deltakit/react";

// Define your custom events
type ReasoningEvent = { type: "reasoning"; text: string };
type ToolCallEvent = { 
  type: "tool_call"; 
  tool_name: string; 
  argument: string;
  call_id?: string;
};
type ToolResultEvent = { type: "tool_result"; call_id: string; output: string };

// Union with built-in events
type CustomEvent = 
  | { type: "text_delta"; delta: string }
  | ReasoningEvent
  | ToolCallEvent
  | ToolResultEvent;
```

### 2. Accumulate Reasoning Chunks

Use `setMessages` to accumulate reasoning into the last part instead of creating new parts:

```tsx
function Chat() {
  const { messages, isLoading, sendMessage } = useStreamChat<ContentPart, CustomEvent>({
    api: "/api/chat",
    onEvent: (event, helpers) => {
      switch (event.type) {
        case "text_delta":
          helpers.appendText(event.delta);
          break;

        case "reasoning":
          // Accumulate into existing reasoning part or create new one
          helpers.setMessages((prev) => {
            const last = prev[prev.length - 1];
            if (!last || last.role !== "assistant") return prev;

            const parts = [...last.parts];
            const lastPart = parts[parts.length - 1];

            if (lastPart?.type === "reasoning") {
              // Append to existing reasoning
              parts[parts.length - 1] = {
                ...lastPart,
                text: (lastPart as { text: string }).text + event.text,
              };
            } else {
              // Create new reasoning part
              parts.push({ type: "reasoning", text: event.text });
            }

            return [...prev.slice(0, -1), { ...last, parts }];
          });
          break;

        case "tool_call":
          helpers.appendPart({
            type: "tool_call",
            tool_name: event.tool_name,
            argument: event.argument,
            callId: event.call_id,
          });
          break;

        case "tool_result":
          helpers.setMessages((prev) => {
            const last = prev[prev.length - 1];
            if (!last || last.role !== "assistant") return prev;

            const updatedParts = last.parts.map((part) =>
              part.type === "tool_call" && part.callId === event.call_id
                ? { ...part, result: event.output }
                : part
            );

            return [...prev.slice(0, -1), { ...last, parts: updatedParts }];
          });
          break;
      }
    },
  });

  // ... render UI
}
```

### 3. Render Reasoning UI

Display reasoning with a "thinking" indicator and proper styling:

```tsx
function ReasoningBlock({ text, isStreaming }: { text: string; isStreaming: boolean }) {
  return (
    <div className="rounded-lg border border-neutral-700 bg-neutral-800/50 p-4 my-2">
      <div className="flex items-center gap-2 mb-2">
        <span className="text-xs font-medium text-neutral-500 uppercase tracking-wider">
          Thinking
        </span>
        {isStreaming && (
          <span className="inline-flex gap-1">
            <span className="w-1.5 h-1.5 bg-neutral-500 rounded-full animate-pulse" />
            <span className="w-1.5 h-1.5 bg-neutral-500 rounded-full animate-pulse delay-75" />
            <span className="w-1.5 h-1.5 bg-neutral-500 rounded-full animate-pulse delay-150" />
          </span>
        )}
      </div>
      <p className="text-sm text-neutral-400 italic whitespace-pre-wrap">
        {text}
      </p>
    </div>
  );
}
```

### 4. Complete Component Example

```tsx
import { StreamingMarkdown } from "@deltakit/markdown";
import { useStreamChat, type ContentPart } from "@deltakit/react";

// Event types
type CustomEvent =
  | { type: "text_delta"; delta: string }
  | { type: "reasoning"; text: string }
  | { type: "tool_call"; tool_name: string; argument: string; call_id?: string }
  | { type: "tool_result"; call_id: string; output: string };

export function Chat() {
  const { messages, isLoading, sendMessage, stop } = useStreamChat<ContentPart, CustomEvent>({
    api: "/api/chat",
    onEvent: (event, { appendText, appendPart, setMessages }) => {
      switch (event.type) {
        case "text_delta":
          appendText(event.delta);
          break;

        case "reasoning":
          setMessages((prev) => {
            const last = prev[prev.length - 1];
            if (!last || last.role !== "assistant") return prev;

            const parts = [...last.parts];
            const lastPart = parts[parts.length - 1];

            if (lastPart?.type === "reasoning") {
              parts[parts.length - 1] = {
                ...lastPart,
                text: (lastPart as { text: string }).text + event.text,
              };
            } else {
              parts.push({ type: "reasoning", text: event.text });
            }

            return [...prev.slice(0, -1), { ...last, parts }];
          });
          break;

        case "tool_call":
          appendPart({
            type: "tool_call",
            tool_name: event.tool_name,
            argument: event.argument,
            callId: event.call_id,
          });
          break;

        case "tool_result":
          setMessages((prev) => {
            const last = prev[prev.length - 1];
            if (!last || last.role !== "assistant") return prev;

            const updatedParts = last.parts.map((part) =>
              part.type === "tool_call" && part.callId === event.call_id
                ? { ...part, result: event.output }
                : part
            );

            return [...prev.slice(0, -1), { ...last, parts: updatedParts }];
          });
          break;
      }
    },
  });

  return (
    <div className="flex h-screen flex-col">
      <div className="flex-1 overflow-y-auto p-4">
        {messages.map((msg) => (
          <div key={msg.id} className="mb-6">
            <p className="text-xs font-semibold text-neutral-400 mb-1">
              {msg.role === "user" ? "You" : "Assistant"}
            </p>
            
            {msg.parts.map((part, idx) => {
              switch (part.type) {
                case "text":
                  return (
                    <div key={idx} className="prose prose-invert prose-sm">
                      <StreamingMarkdown content={part.text} />
                    </div>
                  );
                  
                case "reasoning":
                  return (
                    <ReasoningBlock
                      key={idx}
                      text={part.text}
                      isStreaming={isLoading && idx === msg.parts.length - 1}
                    />
                  );
                  
                case "tool_call":
                  return (
                    <div key={idx} className="my-2 p-3 rounded bg-neutral-800">
                      <p className="text-sm text-indigo-400">
                        Tool: {part.tool_name}
                      </p>
                      {part.result && (
                        <p className="text-xs text-neutral-400 mt-1">
                          Result: {part.result}
                        </p>
                      )}
                    </div>
                  );
              }
            })}
          </div>
        ))}
      </div>
      
      <form
        className="p-4 border-t border-neutral-800"
        onSubmit={(e) => {
          e.preventDefault();
          const input = e.currentTarget.elements.namedItem("message") as HTMLInputElement;
          if (input.value.trim()) {
            sendMessage(input.value);
            input.value = "";
          }
        }}
      >
        <div className="flex gap-2">
          <input
            name="message"
            placeholder="Type a message..."
            className="flex-1 rounded border border-neutral-700 bg-neutral-800 px-3 py-2"
          />
          {isLoading ? (
            <button type="button" onClick={stop} className="px-4 py-2 bg-red-600 rounded">
              Stop
            </button>
          ) : (
            <button type="submit" className="px-4 py-2 bg-neutral-100 text-neutral-900 rounded">
              Send
            </button>
          )}
        </div>
      </form>
    </div>
  );
}
```

## Common Issues

### Duplicate Words

**Symptom**: "TheThe user user is is asking..."

**Cause**: Backend emitting overlapping chunks without deduplication.

**Fix**: Track a buffer and only emit new text:

```python
if text.startswith(buffer) and len(text) > len(buffer):
    new_text = text[len(buffer):]
    yield new_text
    buffer = text
```

### Multiple Reasoning Boxes

**Symptom**: Multiple "Thinking" boxes appearing instead of one.

**Cause**: Creating new reasoning parts for each chunk instead of accumulating.

**Fix**: Check if last part is already reasoning and append to it:

```tsx
if (lastPart?.type === "reasoning") {
  // Append to existing
  parts[parts.length - 1] = { ...lastPart, text: lastPart.text + event.text };
} else {
  // Create new
  parts.push({ type: "reasoning", text: event.text });
}
```

### Reasoning Appears After Response

**Symptom**: Final reasoning box appears after the assistant's answer.

**Cause**: Backend sending complete reasoning event after streaming is done.

**Fix**: Frontend should ignore duplicate complete events if already streamed deltas:

```tsx
case "reasoning":
  // Only add if not already streaming reasoning
  const hasReasoning = last.parts.some(p => p.type === "reasoning");
  if (hasReasoning && event.is_complete) {
    return prev; // Skip complete event, already have deltas
  }
  // ... accumulate as normal
```

## Best Practices

1. **Always accumulate**: Treat reasoning like `text_delta`—accumulate chunks, don't create new parts
2. **Deduplicate backend**: Prevent overlapping chunks from reaching the client
3. **Show streaming state**: Add visual indicator (pulsing dots) while reasoning is active
4. **Preserve formatting**: Use `whitespace-pre-wrap` to maintain line breaks in reasoning
5. **Type safety**: Always define custom event types with TypeScript for compile-time checking
